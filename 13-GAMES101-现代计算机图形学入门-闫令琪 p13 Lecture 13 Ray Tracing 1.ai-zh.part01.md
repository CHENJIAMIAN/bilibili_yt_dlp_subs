#### **2. 光栅化 vs. 光线追踪**

*   **引入光线追踪的原因**：光栅化存在局限性，主要在于难以处理**全局效果**。
    *   **阴影**：
        *   Shadow Mapping方法主要处理硬阴影（点光源）。
        *   虽然有改进方法支持软阴影，但整体实现复杂且困难。
    *   **Glossy（光泽）反射**：
        *   例如打磨光滑的金属（如古代铜镜）。
        *   光线在场景中会发生多次弹射（如：光源 -> 地板 -> 墙壁 -> ... -> 眼睛）。
        *   光栅化处理这种多次弹射非常低效（需多趟渲染），且难以保证物理正确性。
*   **对比总结**：
    *   **光栅化**：
        *   **特点**：快速、近似。
        *   **应用**：实时渲染（如游戏《绝地求生》，帧率需>30fps）。
        *   **局限**：画质相对较低（如无阴影、低模、纹理模糊），因为大地图对GPU要求极高，必须牺牲画质换速度。
    *   **光线追踪**：
        *   **特点**：准确、符合物理规律、质量极高。
        *   **应用**：离线渲染（如电影《疯狂动物城》）。
        *   **局限**：**非常慢**。例如《疯狂动物城》渲染一帧可能需要10,000个CPU小时（即使按1,000小时估算，也远超光栅化）。
*   **核心观点**：图形学中存在大量**Trade-off**（权衡），如MSAA（抗锯齿质量 vs 速度），光线追踪与光栅化亦是如此（质量 vs 速度）。

#### **3. 光线追踪的物理基础与定义**

*   **光线的三个基本假设**：
    1.  光沿直线传播（忽略波动性）。
    2.  光线之间互不碰撞（交叉时互不影响）。
    3.  **光线传播的可逆性**：
        *   光从光源发出，经物体反射进入人眼。
        *   反之，可认为从眼睛（相机）发出光线，经物体反向追踪到光源。
        *   *引用*：尼采——“当你凝视深渊的时候，深渊也在凝视着你。”（寓意：如果你能通过镜子或路径看到对方，对方也能看到你，光路是双向通的）。
*   **历史误区**：
    *   早期科学家（如柏拉图、欧几里得、托勒密）曾认为眼睛会发出“感知光线”接触物体从而产生视觉。
    *   现代仍有约50%的美国大学生支持该错误观点。
    *   在光线追踪中，利用可逆性，我们实际上是从相机发射光线去追踪。

#### **4. 基础算法：光线投射**

这是Whitted风格的基础，用于解决光栅化难以处理的问题。

*   **设定**：
    *   眼睛/相机视为**针孔相机**（一个点）。
    *   光源视为**点光源**。
    *   物体表面发生**完美**的反射或折射。
*   **算法步骤**：
    1.  **发射光线**：对于成像平面上的每一个像素，从相机位置穿过像素中心向场景发射一条光线。
        *   这条光线称为 **Primary Ray** 或 **Eye Ray** 或 **Camera Ray**。
    2.  **求交**：计算光线与场景中物体的交点。
        *   **关键**：只关心**最近的交点**（最近的物体会遮挡后面的物体）。
        *   *优势*：隐式解决了深度测试问题，无需显式的Z-buffer。
    3.  **阴影测试**：从交点向光源连线发射第二条光线。
        *   这条光线称为 **Shadow Ray**。
    4.  **判可见性与着色**：
        *   如果Shadow Ray被遮挡：该点在阴影中。
        *   如果Shadow Ray未被遮挡：该点被光源照亮。
    5.  **计算颜色**：根据光照模型（如Blinn-Phong）计算着色，写入像素。
*   **局限**：光线投射只考虑了一次弹射（光源 -> 物体 -> 眼睛），无法处理镜面反射或折射带来的多次弹射。

#### **5. 进阶算法：Whitted-Style Ray Tracing (递归光线追踪)**

由 **Turner Whitted** 提出，是一个递归算法，旨在解决反射和折射问题。

*   **核心思想**：光线打在物体表面后，会根据材质特性继续分出新的光线进行递归追踪。
*   **算法流程**：
    1.  执行光线投射，找到最近交点。
    2.  在当前交点计算**局部着色**（Local Shading）：
        *   同样发射Shadow Ray判断是否被光源照亮。
    3.  **递归追踪**：
        *   若物体是**反射体**（如镜子）：计算反射方向，发射 **反射光线**（Secondary Ray）。
        *   若物体是**透明体**（如玻璃）：计算折射方向，发射 **折射光线**（Secondary Ray）。
        *   递归调用上述过程，直到达到终止条件（如最大弹射次数或能量耗尽）。
    4.  **颜色合成**：
        *   最终像素颜色 = **局部着色** + **反射光贡献** + **折射光贡献**。
        *   *注意*：需要遵循能量守恒。例如反射占60%，折射占40%，防止颜色无限叠加导致过曝。
*   **光线分类总结**：
    *   **Primary Ray**：从相机发出的第一根光线。
    *   **Secondary Ray**：在弹射过程中产生的光线（反射、折射等）。
    *   **Shadow Ray**：连接交点与光源、用于测试可见性的光线。
*   **效果示例**：
    *   左侧玻璃球（折射）、右侧镜面球（反射）。
    *   能正确生成软阴影（透过玻璃的光线变少）。
    *   性能演变：当年渲染需74分钟（1980年）-> 2006年PC需6秒 -> 现代GPU可实时渲染（几百甚至上千帧）。

#### **6. 技术细节：光线求交**

光线追踪中最核心且频繁的操作是判断光线是否击中物体。

*   **光线的数学定义**：
    *   光线是一条射线。
    *   公式：$r(t) = O + tD$
    *   $O$ (Origin)：起点（如相机位置）。
    *   $D$ (Direction)：方向（单位向量）。
    *   $t$：距离/时间参数，$t \geq 0$。

*   **与隐式表面求交（以球体为例）**：
    *   **球体定义**：球心 $C$，半径 $R$。球面上点 $P$ 满足 $||P - C||^2 - R^2 = 0$。
    *   **求交方法**：
        *   将光线方程代入球体方程：$||O + tD - C||^2 - R^2 = 0$。
        *   展开得到关于 $t$ 的一元二次方程：$at^2 + bt + c = 0$。
        *   利用求根公式 $t = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$。
    *   **解的物理意义**：
        *   判别式 $\Delta < 0$：无交点。
        *   $\Delta = 0$：相切。
        *   $\Delta > 0$：两个交点。
        *   选择 $t$ 最小的正值，即为**可见的最近交点**。
    *   **推广**：对于任意隐式表面 $f(p)=0$，求交即解方程 $f(O+tD)=0$，找最小正实根 $t$。

*   **与显式表面求交（以三角形为例）**：
    *   **重要性**：这是图形学中最基础的操作，因为网格通常由三角形组成。也用于判断点是否在物体内。
    *   **点在物体内判断法（奇偶规则）**：
        *   从该点向任意方向发射一条射线。
        *   统计射线与物体表面的交点个数。
        *   奇数个交点：在物体内；偶数个交点：在物体外。
        *   前提：物体必须是封闭的。
    *   **朴素算法（两步法）**：
        1.  **光线与平面求交**：
            *   平面定义：法线 $\mathbf{n}$ 和平面上一点 $p'$。
            *   平面上任意点 $p$ 满足 $(p - p') \cdot \mathbf{n} = 0$。
            *   代入光线方程 $O + tD$：$(O + tD - p') \cdot \mathbf{n} = 0$。
            *   解 $t$：$t = \frac{(p' - O) \cdot \mathbf{n}}{D \cdot \mathbf{n}}$。
            *   需满足 $D \cdot \mathbf{n} \neq 0$（不平行）且 $t \geq 0$。
        2.  **点在三角形内判断**：
            *   求出交点后，判断该点是否位于三角形内部（可用重心坐标法等）。
    *   **Möller-Trumbore 算法（优化算法）**：
        *   思路：利用重心坐标将“光线与三角形求交”转化为直接解线性方程组，无需先求平面交点。
        *   原理：
            *   交点 $P$ 既在光线上，又可以用三角形三个顶点 $P_0, P_1, P_2$ 的重心坐标 $(u, v, 1-u-v)$ 表示。
            *   建立等式：$O + tD = (1 - u - v)P_0 + uP_1 + vP_2$。
            *   这是一个关于 $t, u, v$ 的线性方程组。
            *   如果解存在且满足 $t \geq 0, 0 \leq u \leq 1, 0 \leq v \leq 1, u+v \leq 1$，则光线击中三角形。
